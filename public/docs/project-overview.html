Request Lifecycle

POST /api/check or /api/ai-check
  ↓
parseCheckRequest()        // 4KB body cap + JSON + Zod validate
isSafeUrlWithDns()         // SSRF: string + DNS (60s cache); blocks rebinding
safeFetch()                // re-validate every redirect
  ↓
/api/check     →  10 checkers via Promise.all → calculateOverallScore + recommendations
/api/ai-check  →  Promise.allSettled([callOpenAI, serperSearch×2]) with AbortSignal threading
                  → parseAIResponse → 7-dimension score (bilingual EN+TH)

ชั้นความปลอดภัย

Security Layers

  • Zod validation + 4KB body cap (DoS + invalid input)
  • SSRF string check — block 127.0.0.1, ::1, AWS metadata, Teredo
  • SSRF DNS check (60s cache) — blocks DNS rebinding
  • safeFetch — re-validates redirect URLs at every hop
  • Response size caps (10MB HTML, 512KB Serper) — memory exhaustion
  • Rate limiter (10/min main, 3/min AI) + CORS allowlist (CSRF / SSRF-relay)
  • sanitizeMeta() + system-message framing — LLM prompt injection
  • AbortSignal threading from request → OpenAI/Serper — cancels paid calls on client disconnect

หมายเหตุสำคัญ: rsync ต้องใช้ --delete สำหรับ .next/static เพราะแต่ละ build สร้าง hashed chunk ใหม่ (ไม่งั้นจะ 404), และต้อง set -a; source .env; set +a ก่อน pm2 restart --update-env เพราะ Next.js standalone ไม่ auto-load .env

Notes: rsync needs --delete on .next/static because each build creates new hashed chunks (otherwise browser requests get 404), and you must set -a; source .env; set +a before pm2 restart --update-env because Next.js standalone doesn't auto-load .env

08โปรเจกต์นี้ถูกสร้างขึ้นมาได้ยังไงHow This Was Built

aicheck เขียนด้วยวิธี "Vibe Coding" — สั่งงาน AI ด้วยภาษาพูด Developer เป็นผู้กำกับ (สั่ง/ตรวจ/ตัดสิน architecture) AI เป็นผู้เขียนโค้ด

aicheck was built with "Vibe Coding" — instructions in plain language. Developer directs (commands, reviews, decides architecture); AI implements.

Toolchain: Claude Code CLI (Opus 4.6, 1M context) ใน VS Code integrated terminal · Git + GitHub (AI ทำ branch/commit/PR ได้แต่ human review ก่อน merge) · Playwright MCP สำหรับให้ AI ควบคุม browser จริงตอน test

Toolchain: Claude Code CLI (Opus 4.6, 1M context) in VS Code's integrated terminal · Git + GitHub (AI handles branches/commits/PRs, humans review before merge) · Playwright MCP so the AI can drive a real browser during testing

Subagents ตรวจคุณภาพโค้ด (4 ตัว — รัน parallel ผ่าน /review-all)

Code Review Subagents (4 — run in parallel via /review-all)

  • security-auditor — SSRF, XSS, injection, rate limiting, input validation, prompt injection
  • performance-error-reviewer — memory leaks, timeouts, algorithm complexity, try-catch coverage
  • react-typescript-reviewer — accessibility, memoization, type safety, re-render issues
  • maintainability-reviewer — naming, dead code, duplication, "human reader" test

ทุก agent อ่าน .known-issues.md ก่อนเสมอ เพื่อ skip findings ที่ทีมยอมรับแล้ว (เช่น CSP unsafe-inline, rate limiter in-memory) — ป้องกันการ flag ซ้ำทุกรอบ Phase 2 ของ /review-all ทดสอบกับเว็บจริงเพื่อจับ semantic bugs ที่ static review ไม่เห็น

Every agent reads .known-issues.md first to skip findings the team has explicitly accepted (e.g., CSP unsafe-inline, in-memory rate limiter), preventing repeated re-flagging. Phase 2 of /review-all tests against real websites to catch semantic bugs static review can't see.
